Start Code for JPA and JAX-RS projects with support for OpenAPI Documentation

API-Documentation

Scrum Plan

Sprint 1

  • The API description must be almost complete and available as a page on your deployed project.
    • The base skeleton of the API description
  • The Scrum Plan for this, and the coming sprints.
    • As a page on the deployed project.
  • The Travis build-pipeline must be setup.
  • The base entities and their relations, as well as their facades, must be implemented.
    • Using TDD.
  • Setup OpenAPI skeleton

Sprint 2

  • More advanced entities + relations + facades should be completed
    • Using TDD.
  • Sample data should be available in the dev-database.
    • Using a base Tester class
  • Some of the endpoints (GET, POST and PUT) must be ready with the corresponding DTOs and integration tests

Sprint 2

  • API must be fully implemented
  • Implement a SPA which as a minimum must have the ability to use some of your GET endpoints and at least one POST endpoint.
    • Get all persons with a given hobby
    • Get all persons living in, a given city (I.e. 2800 Lyngby)
    • Get the count of people with a given hobby
    • Get a list of all zip codes in Denmark
    • Get a list of companies with more than x employees
    • Create a person, with hobbies, phone, address, etz.
